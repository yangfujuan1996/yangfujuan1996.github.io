python爬虫基础

一、网络原理
1、HTTP与HTTPS的区别
HTTP：我们成为超文本传输协议，主要用来实现网络传输超文本到本地浏览器，实现高效且准确的超文本传输。目前HTTP版本为1.1
HTTPS：我们认为就是以安全为目标的HTTP通道，即在HTTP协议的基础上加入了SSL层，简称HTTPS。
SSL的作用：
第一、为HTTP传输文件提供安全的传输通道
第二、确认网站的真实性，通过HTTPS访问的网站都需要获取到CA认证。
2、OSI七层模型：
第七层应用层
协议：DHCP • DNS • FTP • Gopher • HTTP • IMAP4 • IRC • NNTP • XMPP • POP3 • SIP • SMTP • SNMP • SSH • TELNET • RPC • RTCP • RTSP • TLS • SDP • SOAP • GTP • STUN • NTP • 更多
第六层表示层
不用协议
第五层会话层不用协议
第四层传输层
协议：TCP • UDP • DCCP • SCTP • RTP • RSVP • PPTP • 更多
第三层网络层
协议：IP (IPv4 • IPv6) • ARP • RARP • ICMP • ICMPv6 • IGMP • RIP • OSPF • BGP • IS-IS • IPsec • 更多
第二层数据链路层
协议： 802.11 • 802.16 • Wi-Fi • WiMAX • ATM • DTM • 令牌环 • 以太网 • FDDI • 帧中继 • GPRS • EVDO • HSPA • HDLC • PPP • L2TP • ISDN • 更多
第一层物理层
协议： RS-443 、RS-232C、RS-485 、理-2593
3、HTTP的请求机制
第一、有客户端（浏览器）发出请求
第二、通过网络传输，到指定的服务器地址
第三、服务器接收相关请求并进行解析
第四、服务器将最终结果响应回客户端
第五，客户端接收到服务器响应后，通过浏览器解析，最终将内容进行展现
4、客户端访问服务器，发出请求，有四部分构成
第一，请求方法（request method），默认为get方式。只有在表单提交和文件上传时为post方法。
第二，请求网址（request url），即在浏览器中输入的网路地址。
第三，请求头（request header），用来标识为一个浏览器正常的访问
第四，请求体（request body），用来传输到服务器端的相关数据
5、请求方法中get与post的区别
第一、使用get方式提交请求时，相关数据会作为url的参数被一起进行提交，我们可以直接在地址栏中观察到相关数据。而采用post方式请求的url不会包含这些数据，而是直接通过表单提交到服务器。
第二、get方式提交请求，对于提交的数据大小不能大于1024字节，而post方式提交数据，没有大小的限制。
总之，如果提交的信息比较敏感或者字节较大时，需要使用post方式；如果只是普通的请求使用get方式。

1、概念：迭代器访问元素是从第一个元素开始，依次访问后续元素直到结束。其中迭代器只能向前不能回退。其中 包含两个方法iter()和next()。
Iter()用来创建一个迭代器，next()用来访问下一个元素。
字符串、列表、元组都可以创建迭代器，他们也是可迭代的对象。

迭代器两个步骤：
第一、当调用next()函数，会将迭代器当前位置进行改变
第二、将下一个元素进行返回
迭代器的延迟作用：迭代器生成的元素并不是在迭代器声明时就准备好了的。通常在调用迭代器时才会生成对应元素，不需要时并不会主动地生成相关元素。
我们可以把迭代器看成一个容器，到我们需要时使用next()方法返回容器中的下一个值。
迭代器实现了工厂模式的对象，它在你每次访问下一个值的时候给你返回。

三、生成器
1、创建方式

二、局部变量与全局变量
1、在Python中，函数在调用时才会将函数载入，给函数一个内存地址，用来执行内部的相关代码。（在载入时，先载入全局变量，后载入局部变量；在函数内调用时，先调用局部变量，在调用全局变量。）
2、Python程序在运行时，首先会将最外部的变量载入内容，并给其分配相关的内存空间。称这种变量为全局变量。
3、函数中声明的变量，只有在函数被执行时，才会被分配内存空间进行执行。称这种变量为局部变量。
4、Python程序执行过程中，执行函数时首先检查函数内部的局部变量是否存在，如果存在就调用它，如果不存在就从外部全局变量中进行查找，若存在就调用它，若不存在，就报错。

6、当函数出现嵌套情况时，最内层函数执行调用变量，会采用“就近原则”即先查找自身范围内的变量，如果没有找到，则向上查找，直到程序最外部结束。

最后，解决了在同一个文件中，方法和变量重名的问题。
3、导入时的问题
如果导入了与文件同名的函数（方法名相同，函数个数相同）时，需要使用模块名调用函数，与文件中的函数进行区分，例如：

4、模块可以在Python文件中，实现多次import导入，但是只有第一次导入时才真正将模块中的内容运行、方法载入，其他再进行导入时，Python解析器会先检测是否已经载入该模块，如果载入就不在执行相关操作。
5、总结：
模块载入一共分三个步骤：
1）为模块的导入，创建命名空间（划分新的内存地址），被载入的文件按照全局作用域来使用。
2）模块载入后，将在上一步操作中创建的命名空间中，执行模块中的代码
3）创建一个模块的名字，来引用模块所在的命名空间。my.show()
6、当使用from import引入多个模块时，同时多个模块存在同名函数，那么在程序中调用该函数时，以程序导入的最后一个模块的函数为准。

3、类变量与成员变量的区别
1）类变量是声明在类中，位于函数之外声明的变量。这些变量是公共的，即在类外使用类名.类变量的方式进行调用。我们一旦修改类变量的值，所有调用类变量的内容均发生改变。
2）成员变量，是声明在类中的构造函数中，通过self.变量来进行声明赋值的。这些成员变量是公有的，即在类外可以通过对象名.成员变量的方式进行调用。所有的对象都有自己的成员变量，也就意味着每个对象的成员变量都是独立的，不会相互干扰。

4）成员变量和方法，默认情况下都是公有的。也就是说在类外可以直接使用对象进行调用和修改，因此公有的是不安全的。我们可以通过为成员变量设置双下划线，将其标识为私有的成员变量，例如：self.__name = ‘tom’。私有的成员变量只能在类内进行直接使用，类外必须通过相关方法进行间接调用。

二、继承
1、Python中能够实现单继承与多继承。
单继承：就是子类只有一个直接父类。
多继承：子类有两个及两个以上的直接父类。
2、子类继承父类，子类可以继承父类的相关特征。子类继承了父类的方法，如果在多继承情况下，父类出现同名函数，那么子类继承该函数时，根据继承的顺序进行调用。
3、super()函数，它指代了当前类的父类。我们可以在子类中调用super函数，用来初始化父类。但是使用super()调用父类的构造函数时，只能对多继承中的第一个父类生效。
4、抽象与继承的区别与关系
抽象：指对同一类别事物的相似部分进行抽取。抽象我们认为是对具体化对象的相关行为和属性的泛化过程。
继承：Python中我们通常认为类就是一个抽象的内容。类本身并不能直接执行，需要通过继承，进行对象的实例化，进而获得具体的对象本身，才能执行相关的功能。
也就是说继承其实是抽象的具体化过程。
5、多继承中，当一个子类继承多个父类时，我们需要如下声明：
class SubClass(A,B):
其中，A、B是SubClass的父类。那么，如果我们分为两种个情况：
第一，子类中没有显式的定义init()，那么在实例化子类时调用其父类的构造函数。如果有多个类，都定义了init()函数，那么它会调用第一个具有init__()函数的父类。
第二，子类中显式的定义理论init_(),那么实例化子类时不再调用父类的构造函数。此时，如果我们要调用父类的构造函数，有两种方法。
1）在子类的构造函数中，显式的调用对应父类的构造函数，例如：
A.init__()
2)采用super()函数，super().init()或者super(子类，self).init()

五、classmethod方法
在Python中，我们通常使用classmethod用来修饰一个方法，这个方法就被称为类方法，类方法通常用来操作类变量。类方法和类变量一样，都需要通过类名来调用。

六、staticmethod方法
在Python中，我们使用staticmethod来修饰函数，该函数被称为静态方法，首先，静态方法不需要使用对象参数或类参数作为默认参数。其次，静态方法通常只能调用类变量，而不可以使用成员变量。最后，在类外进行调用时，可以使用类名来调用。

实例方法、类方法与静态方法的区别
实例方法：第一个参数必须是实例对象（self）,通过它来传递实例的属性和方法，只能由实例对象来调用。
类方法：使用装饰器@classmethod，第一个参数必须是类对象（cls）,通过它来传递类的属性和方法。可以通过实例对象或类对象来调用，通常使用类对象调用。
静态方法：使用装饰器@staticmethod,没有默认参数（self和cls），但是方法体中不能使用类或实例对象的任何属性和方法。可以通过实例对象或类对象进行调用。

1、类方法和静态方法都可以通过对象实例或者类名来调用。
2、类方法主要针对的是类，可以通过继承和重新定义
3、静态方法不能被继承，无法被重写，可以被认为是全局函数。静态方法在使用时完全不会用到对象实例本身。静态方法逻辑上属于类，但是不会涉及到类中的属性和方法的操作。

七、方法重写
概念：当子类继承父类时，子类会继承父类所有的公有方法。如果在子类中重写定义一个与父类中同名的方法，我们成为方法的重写（覆盖）。方法重写只需要名字相同即可，与方法的参数个数、顺序无关
八、多态与鸭子类型
多态：多态一般都是通过继承来实现的，也就是根据不同的对象形态来进行区分。我们在Python中，使用多态时，意味着在编码时变量并不知道所引用的对象具体内容，只有在程序运行时相关对象被实例化后，才能够确定。
鸭子类型：在Python中，因为其自身特性并没有对多态进行明确的规定，可以创建一个功能相似，但与原对象无关的、全新对象，来实现程序的松耦合。
我们通常在开发中，采用接口的方式来实现。
我们认为在一组功能相似，但没有继承关系的对象中，采用接口方式。
